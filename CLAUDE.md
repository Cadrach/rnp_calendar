## Project structure

```
rnp_calendar/
├── client/   React + Vite + TypeScript frontend
└── server/   Laravel 12 API-only backend
```

See `SPECS.md` for full feature documentation.

---

## Backend (server/)

- **Laravel 12**, PHP 8.2, SQLite, Composer
- API-only: no Blade, no Vite, no frontend assets
- **Auth:** session-based, `web` guard — no Sanctum, no tokens
- **Routes:** `routes/api.php` only
- Run migrations: `cd server && php artisan migrate`

### Key services

- `DiscordClient` — wraps Discord API v10 (get/post/patch, forum threads, members, roles)
- `EventDiscordSync` — syncs events to the SÉANCES Discord forum channel on every create/update

### Config

Discord IDs live in `config/services.php` under `services.discord.*`, sourced from `.env`:

- `DISCORD_BOT_TOKEN`, `DISCORD_GUILD_ID`
- `DISCORD_CHANNEL_ID_PROPOSITIONS`, `DISCORD_CHANNEL_ID_SEANCES`
- `DISCORD_ROLE_ID_MJ`, `DISCORD_ROLE_ID_ADMIN`

## Frontend (client/)

- **React 18**, Vite, TypeScript, Yarn
- **UI:** Mantine 8
- **Data fetching:** TanStack React Query
- **Routing:** React Router v7
- **API client:** generated by Orval from OpenAPI (Scramble) — files in `client/src/api/generated/` are auto-generated,
  do not edit them manually
- **ALWAYS use generated hooks** — never use `axiosInstance` or do fetches directly. If an endpoint doesn't have a
  generated hook, regenerate the API client first (`php artisan scramble` + `yarn orval`)
- Custom hooks that Orval cannot generate go in `client/src/api/` (e.g. `event-register.ts`) and must follow the same
  shape as Orval-generated hooks
- Type overrides for generated models go in the consuming file (e.g. `DictionaryContext.tsx`), not in the generated
  files — the linter reverts those

### Key patterns

- Dictionary data (games, rooms, scenarios, members, current user) is loaded once via `DictionaryContext` and consumed
  anywhere with `useDictionary()`
- `DiscordMember` interface is defined in `DictionaryContext.tsx` and re-exported from there
- Register/unregister hooks (`useEventsRegister`, `useEventsUnregister`) are in `client/src/api/event-register.ts`
- After any mutation that changes an event, invalidate **both** the list query key and the show query key

## Conventions

- French locale for dates in Discord thread content (Carbon `->locale('fr')`, `isoFormat`)
- Discord thread starter message ID = thread ID (used for `editMessage` on forum posts)
- `flags: 4` (SUPPRESS_EMBEDS) on all Discord messages and thread creation
- Soft deletes on events
- `player_ids` stored as a JSON array of Discord IDs (strings)
- MJ/Admin role checks are computed attributes on the `User` model (`is_mj`, `is_admin`) based on the `roles` JSON
  column
- The `discord_members` cache key is cleared when a new user is created on first login

## Dev workflow

```bash
# Backend
cd server && php artisan serve        # http://127.0.0.1:8000

# Frontend
cd client && yarn dev                 # http://localhost:5173

# After backend model/endpoint changes, regenerate the API client:
cd server && php artisan scramble     # regenerates openapi.json
cd client && yarn orval               # regenerates client/src/api/generated/
```
